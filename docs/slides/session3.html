The Agent Loop

What Happens When You Ask a Question?

1

System Prompt

Schema description tells the LLM what tables and columns exist

2

Tool Call

LLM writes SQL (or Python) and requests execution

3

Execute & Return

System runs the code and sends results back to the LLM

4

Review & Deliver

LLM checks results, fixes errors or delivers the answer

The Key Insight: Test and Review

The agent doesn’t just generate code and hope for the best. It writes code, executes it, reviews the result, and decides whether to deliver or try again.

This is the same pattern a careful analyst follows: write a query, check the output, refine if something looks off, then present the final answer. The LLM automates that entire cycle.

The Agent Loop in Pseudocode

agent_loop.py — The Core (~50 Lines)

def agent_loop(question, schema):
    messages = [system_prompt(schema), user_message(question)]

    for attempt in range(3):
        response = llm.chat(messages)
        if response.has_tool_call:
            result = execute(response.tool)
            messages.append(result)        # feed back
        else:
            return response.text           # done

The System Prompt Is the Secret

Why the Prompt Matters

Without Join Key Documentation

  • LLM sees orders and customers tables
  • No information about shared keys
  • Agent can’t figure out how to merge
  • Query fails or returns wrong results

With Join Key Documentation

  • Prompt says: orders.customer_id links to customers.id
  • Agent writes correct JOIN
  • Merges work on first attempt
  • Accurate, complete answers

The Prompt Is “Institutional Knowledge”

The system prompt encodes everything the agent needs to know about your data: table names, column meanings, relationships, business rules, and formatting preferences.

Change the prompt, change the behavior. Add a rule like “always filter to active customers” and every query respects it. Remove join documentation and merges break. The prompt is the most important piece of the system.

Where Does the Code Run?

Three Architecture Options

A. Cloud LLM Sandbox

  • Vendor runs the code (OpenAI, Anthropic)
  • Simplest to set up
  • Data leaves your network
  • Fine for non-sensitive data

B. Your Own Containers

  • Docker on your servers or cloud
  • Production answer for sensitive data
  • You control the execution environment
  • More infrastructure to manage

C. SQL-Only Mode

  • No Python execution at all
  • LLM generates SQL, your DB runs it
  • Safest for regulated environments
  • Can’t do in-memory merges or charts

Choosing the Right Option

Cloud Sandbox Your Containers SQL-Only
Setup effort Low Medium Low
Data stays on-prem No Yes Yes
Python/charting Yes Yes No
Best for Prototyping Production Regulated

What the Other 2,950 Lines Do

Security & Access

  • Authentication (who are you?)
  • Authorization (what can you see?)
  • Row-level access control
  • Audit logging

Reliability & Scale

  • Error handling & retries
  • Rate limiting
  • Multi-tenancy
  • Monitoring & alerting
  • Failover & backup

The core loop is elegant and simple. The production wrapper is where the engineering effort lives. This is why you buy or build on a platform rather than rolling your own from scratch.
