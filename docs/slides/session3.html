How RAG Works

1

Chunk

Documents split into passages (paragraphs, sections, or clauses)

2

Embed

Each chunk converted to a numerical vector by an embedding model

3

Store

Vectors stored in a vector database for fast similarity search

4

Query

User’s question is embedded and matched against stored chunks

5

Generate

Top matches sent to LLM as context; LLM answers with citations

RAG = Retrieval-Augmented Generation. The LLM does not memorize your documents — it searches them at query time and answers using only what it retrieves.

Say: “Let me pull back the curtain on how that document Q&A actually works. The technique is called RAG — Retrieval-Augmented Generation. Here is the pipeline in five steps.”

Walk through each step: “First, your documents are split into chunks — paragraphs, sections, or clauses. Second, each chunk is converted into a numerical vector by an embedding model — essentially, a mathematical fingerprint of the text’s meaning. Third, those vectors are stored in a vector database optimized for similarity search.”

“When a user asks a question, their question is embedded the same way and matched against stored chunks — step four. The closest matches are retrieved. Step five: those matches are sent to the LLM as context, and the LLM generates an answer using only the retrieved passages, with citations back to the originals.”

“The key concept here is chunking — how you split your documents determines what the AI can and cannot find. Chunk too large and retrieval is noisy. Chunk too small and you lose context. We will see a chunking failure in the Atherton case you will read before Session 6 — a $1.5 million contract clause was split across two chunks, and the second half fell below the retrieval threshold. The agent returned a confident, well-cited answer that was missing a critical penalty. That is the kind of failure you need to know about.”

“For now, the essential takeaway: the LLM does not memorize your documents. It searches them at query time and answers using only what it retrieves.”
