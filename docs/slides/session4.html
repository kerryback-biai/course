Anatomy of a Reporting Pipeline

Each Step Is Customizable

Data Steps

  • Query — which systems, which tables, what date range
  • Clean — handle nulls, standardize formats, deduplicate
  • Merge — join across systems on shared keys
  • Compute — KPIs, YoY change, variance to budget

Output Steps

  • Chart — bar, line, waterfall, heatmap; style and palette
  • Narrate — tone, length, audience, what to emphasize
  • Assemble — layout, headers, appendix, export format
  • Review — human-in-the-loop checkpoint before distribution

The pipeline is the same agent loop from Session 3, executed multiple times in sequence. Each step’s output feeds into the next step’s input.

The Art of Iteration

Prompt Specificity Drives Quality

Generic Prompt

  • “Summarize Q3 results”
  • Gets a wall of text
  • No structure, no emphasis
  • Fine for a first draft

Structured Prompt

  • “Use traffic-light format: green/yellow/red for each division”
  • Clear visual hierarchy
  • Easy to scan
  • Much more useful

Executive Prompt

  • “One page, lead with biggest risk, include 90-day action items”
  • Board-ready output
  • Prioritized by impact
  • Actionable recommendations

Iteration in Practice

Draft 1: “Summarize quarterly performance.” — You get a generic overview.

Draft 2: “Use traffic-light format. Flag any metric more than 10% off plan.” — Now it’s scannable.

Draft 3: “One page maximum. Lead with the biggest risk. End with three recommended actions.” — Board-ready.

Each iteration takes seconds. The cost of refining is near zero. The skill is knowing what to ask for — and that’s a business skill, not a technical one.

Hands-On Exercises

Exercise 1: Customer Risk Report

Exercise 1 — Customer Risk Report

  • Query accounts receivable aging and customer order history
  • Flag customers with invoices over 90 days past due
  • Cross-reference with recent order volume (are they still active?)
  • Output: one-page risk summary with traffic-light ratings

Exercise 2: Division P&L Comparison

Exercise 2 — Division P&L Comparison

  • Pull revenue and expense data for 3 divisions
  • Compute gross margin, operating margin, and YoY change
  • Generate a comparison chart (grouped bar or waterfall)
  • Write a two-paragraph narrative highlighting key variances

Exercise 3: Board-Ready Memo

Exercise 3 — Board-Ready Memo

  • Combine data from Exercises 1 and 2
  • Add headcount and pipeline data
  • Structure as: Executive Summary, Key Risks, Division Performance, Outlook
  • One page. Lead with the single most important thing the board needs to know.
  • Iterate at least twice on tone and structure

From Ad Hoc to Automated

Same Pipeline, Different Trigger

Ad Hoc (Chat)

  • User types a request
  • Agent runs pipeline
  • Delivers to the user
  • Great for exploration and one-offs

Automated (Scheduled)

  • Cron job triggers the pipeline
  • Monday morning dashboard refresh
  • Monthly board deck generation
  • Human review gate before distribution

The architecture is identical. The only difference is who triggers it — a person typing in chat or a scheduler running on a timer.

Case Study: HPE

HPE: “Alfred” Replaces 100-Slide Weekly Review

CFO Marie Myers deployed an AI agent called Alfred to automate Hewlett Packard Enterprise’s weekly performance review. The 100-slide deck that took a team days to prepare is now generated automatically — with humans reviewing the output, not creating it.

— HPE executive communications, 2025–2026; reported by Fortune and CFO Dive
