## Database: Superstore Sales

Table: `superstore` (9,994 rows)

Retail orders from a US office supply chain, 2014-2017.

| Column | Type | Description |
|--------|------|-------------|
| "Row ID" | BIGINT | Unique row identifier |
| "Order ID" | VARCHAR | Order identifier (e.g., CA-2017-152156) |
| "Order Date" | DATE | Date the order was placed |
| "Ship Date" | DATE | Date the order was shipped |
| "Ship Mode" | VARCHAR | Standard Class, Second Class, First Class, Same Day |
| "Customer ID" | VARCHAR | Customer identifier |
| "Customer Name" | VARCHAR | Customer full name |
| Segment | VARCHAR | Consumer, Corporate, Home Office |
| Country | VARCHAR | Always "United States" |
| City | VARCHAR | City name |
| State | VARCHAR | US state name |
| "Postal Code" | VARCHAR | ZIP code |
| Region | VARCHAR | East, West, Central, South |
| "Product ID" | VARCHAR | Product identifier |
| Category | VARCHAR | Furniture, Office Supplies, Technology |
| "Sub-Category" | VARCHAR | 17 sub-categories (Phones, Chairs, Storage, Tables, etc.) |
| "Product Name" | VARCHAR | Full product name |
| Sales | DOUBLE | Revenue in USD |
| Quantity | BIGINT | Number of items |
| Discount | DOUBLE | Discount rate (0.0 to 0.8) |
| Profit | DOUBLE | Profit in USD (can be negative) |

**Note:** Column names with spaces must be quoted in SQL: `SELECT "Order ID", Sales FROM superstore`

**Key relationships:**
- Each order can have multiple rows (one per product)
- Profit can be negative (discounted or loss-making items)
- Discount of 0.0 means no discount
